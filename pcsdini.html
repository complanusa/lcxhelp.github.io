PCSD.INI Reference

Introduction

The file pcsd.ini defines the structure and contents of the menus that WinPCS runs. The name is historical: back in the very early days, when what became WinPCS was a collection of executable programs, the main part was PCSD.EXE, which with its configuration file would allow opening of View and Entry screens (EIVIEW.EXE and EIEN.EXE).

Like all ini files, this file contains a number of sections, with entries in them. The section-names generally correspond to the drop-down menus, and the entries within the sections become the entries in the menus.

Fixed parts of the menu

The dropdown menus System, Edit, Window, and Help, are standard and hardcoded, and for the most part are not controlled by the entries in PCSD.INI. However, the System/Run sub-menu can be populated from the [Run] section in pcsd.ini, and there can be additional items added to the Help menu from entries in a section [Help].

The presence and absence of other items in the System and Edit menus can be controlled wholesale via entries in the [MENU] section in winpcs.ini.

All other sections in the pcsd.ini file become top-level or sub-level drop-down menus.

Entries format

All menu entries have the same general format, beginning with the text to be shown as the menu item as the name of the entry, much like the section-names also are the texts shown for the drop-down menus. 

text=file action arguments
text=file action(icon) arguments

The file is a reference to a file, this is the INI file for a View or Entry, an LCX-file for running LCX functions, or the help-file or executable file to be opened and run. What is to be done with the file is determined by the action value. An optional icon to be shown in the menu, making it prettier and more user-friendly, is then included in a set of parentheses right after the action code. Following this are additional required arguments for the particular action: Views have to refer to some table, and LCX-calls have to identify the function to be run.

The following action-codes are defined and currently in use:

ActionDescription Arguments Example
LRun an LCX function LCXfunction, then any string to be passed to the function LCX Info=0:\openfile.lcx L(WINPCS) ShowHTML http://lcxhelp.com
LaRun an LCX function in a separate thread LCXfunction, then any string to be passed to the function Tags without check-sheets=0:\tagcheck.lcx La(BMTAG) TagChecks
L/Run the LCX function in response to the user typing the forward slash, for the purpose of searching. LCXfunction, then any string to be passed to the function Search for Item=0:\winpcs.lcx L/(CORE) FindTag2
VOpen a View on some table The table to be Viewed. Master Equipment=6:VEQUIP.INI V(ENGEQUIP) ENGEQUIP

Addtitionally, these actions were once defined but have fallen out of use, since the LCX functions are much more flexible:

ActionDescription
EOpen Entry on some table, syntax is similar to the one for Views.
HOpen a Help file. No arguments. Obsolete as we are no longer writing .hlp files.
XRun some executable file.

The file-reference can be, and usually is, given with the numbered drive, as defined in CLIENT.INI, so the actual location in the file-system at any one time will not matter. See ExpandPath( ) for details.
