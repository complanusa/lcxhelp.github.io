Starep.ini file reference

File Date: 2022-04-05

Each of the sections in this file refers to a report layout. These are referenced from records in REFREP, grandrep.ini, or the starepref argument to the functions that generate status index reports.

Entries in the section are of several types: fields of the report parameter settings postprocess actions pushbuttons selection spec

Fields

table field function=width (legend) (aname)

Special value:
$leaf=width (legend) (aname)
equivalent to
#leaf #itemfield=width (legend) (aname)

table is actual tablename or #n for levels, #penult, #carrier, or #leaf for the lowest levels.

field is actual fieldname or #cxdictname for standard fields (see CXDICT documentation and the function StdField( ))

function is optional LCX function to be called as per on-field functions, (category 9)

width is the visual width of the field

(legend) is the legend to be shown. In parentheses, so it allows spaces

(aname) is the optional alias name for the field. Use with RepInfo("alias aname", identifier);

Parameters

$param name=value

Parameters are subsequently read through calls to RepInfo("param name", identifier);

Pushbuttons

$button text=action 0 (info) arguments

$button icon=action 1 (info) arguments

Specific actions:

$button text=QREPORTsuf 0 (Run a Quantitative summary report) Qreport S: L: G: T:

See the pushbutton references for Qreports in qreportini.html for more details on this.

$button text=LCXCALL 0 (Do something) LCXfunction argument-tail

$button text=LCXCALLA 0 (Do something if allowed to) LCXfunction argument-tail

See Category 6 #4 actions for the details on these functions.

Selection spec

$select lilprint Print!=1 (Print Y/N) I:3 Y N #carrier RECEIVED PrintOut

$button Print!=FORALL2 0 (Print Selected Records) function args

See more details in the docs on the forall functions, category 13

Postprocessing

$postprocess=CMI+L:#0+L:#1+P:ASORT
