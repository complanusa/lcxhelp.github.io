Information on the SICLIENT utility

File Date: June 17 2003

SICLIENT

The SICLIENT utility allows quiet reading of status index information, and generates a large CSV file with the results. The program is a command-line .EXE file, which uses a number of the standard client-side DLLs and information to connect to the server.

Command-Line Parameters

The command-line is:
siclient [options] starttable reportkey startvalue

where the options, which also may appear between or after the three parameters, can be any combination of:

-c csvfile emit output into a CSV file. If none is specified, the data will be emitted in CSV format to standard output.
-q quotechar specify alternate quote character (default is ")
-s sepchar specify alternate field separator character (default is ;)
-D systemdir specify where to look for ANCHORC.DLL The default for this is the current directory
-U username username to use for WinPCS. The default is the same as is used on the network. WinPCS may ask for a password if so configured.
-S number This is the number of the server to use from WinPCS.INI
-v number Indicate level of verbosity. (default 0 means quiet)
-r startvalue is the record number (eg. RSERIAL) as opposed to a general string (eg. SYSTEM_NO)

The starttable, reportkey, and startvalue are as follows:

starttable Table on which to start the report
reportname Name of report-template in REFREP The combination of these becomes the REFREP key
startvalue System-number of object to start the report on, or RSERIAL of the record, as per the -r option

The resulting report can be read by other programs that can grab the CSV file, or post-processed by programs such as SIPOST or other utilities.

Example:


siclient -D ..\system32 -S 1 -U knut CERTSITE A11 -r 1 -c sp11.csv -v 2

This runs siclient report A11 (as per the REFREP key) for table CERTSITE, record number (RSERIAL) 1. The output CSV file is sp11.csv, which can be re-read into a WinPCS report-view through SYNFARE.

Postprocessing

SIPOST and CHART.HTML

One of the most popular reports are the succinct high-level summary reports from a project. The SIPOST program takes the CSV file produced by SICLIENT, and allows certain kinds of summarizations to be run on it. SIPOST then writes a short file in JavaScript format, that can be used as input to a graphing program run within a browser.

Command-Line Parameters

The command-line is:
sipost [options] datafile

where the options, which also may appear after the datafile argument can be any combination of:

-a name append output to the end of the named file
-o name overwrite output to the named file The default is to overwrite a file called data.js
-f column Column index of the filter column, typically System
-m name Value to match for the filter column. If the -f or -m options are omitted, the data summary will be over the entire data set from the CSV file.
-t column Column index of the text column, typically Discipline
-s column Column index of the sum column, typically Status. Column indexes start at 0 for the first column, 1 for the second one and so on.

Example:


sipost sp11.csv -t 10 -s 6 -o data.js
