WinPCS.INI Client configuration file reference.

File Date: 2014-04-02

This references the configuration file for the client. Like other .ini files, this file has a number of sections with entries in them, and the sections may appear in any order.

The names of the sections and the keywords within the sections are generally case-insensitive. Values of entries, to the right of the equal sign, may be case-sensitive or not depending on the specific situation.

The sections are:
[WinPCS]
[MENU]
[STRIP]
[PATHS]
[SYNFARE]
[PPVIEW]
[HIER]
[VIEW]
[REPORT]
[RCONN]
[HIERTOP]

[WinPCS]

Several miscellaneous options that do not require their own section are defined here.

The entries in this section are:

server= Server address
endpoint= TCP Port number to be used
defgroup= Database-system to be used
dynamicserver= TCP dynamic server handling
servertxt= TCP Description of server
proxy= TCP Proxy setting
heartbeat_ival_proxy= TCP Heartbeat setting for proxy
heartbeat_ival_direct= TCP Heartbeat setting for direct connection
auth= DCE/RPC authenthication setting
speedtest_interval_ms=
speedtest_latest=
speedtest_logfile=		 Check network speed
iolog= Log activity between client and server to a file.
interval= Server presence Checking interval
periodfiles= Client-side periodic activity setting
periodcall= Client-side periodic activity setting
rightmenu= Right-click menus origin
lockout= Lock the client or turn on a message of the day
editor= Default preferred text editor
ddesvc= List of known dde servers
debug= Trace-printing and Debugging target
cdatdebug= Trace-printing and Debugging target for CDAT interactions
library= Pre-loaded LCX files
appearance= Variations on the way Views, Entry, and Reports look.
transferblock= Number of bytes per turn when transfering files to or from the server.
onstart= A function to be run when WinPCS starts.
startfiles= Files where the onstart= function and any supporting functions are defined.

server=

This entry indicates the name of the server where the WinPCS server is expected to be found. Alternate servers, numbered 1, 2, etc. can be identified as SERVER1=, SERVER2= etc. These are referred to via the command-line argument -S.

endpoint=

For TCP and for DCERPC, the server is set to listen on some TCP port using either the WinPCS protocol or the ncacn_ip_tcp RPC protocol. The client must attempt to connect to the server on the same port number. The default values are set in ANCHORS.DLL and ANCHORC.DLL, but if some other port number is used it can be specified here.

Different endpoints for internal and external server addresses can be specified as endpoint1= endpoint2= etc for the correspondingly numbered server, if the default from anchorc.dll or the specified non-numbered endpoint= is not the right one.

A matching setting for the endpoint should be present on the server.

The default is the port number as defined as the Endpoint in the ANCHORC.DLL.

The IANA has allocated port 5166 as available for WinPCS. This port number is usually what is set as the default in the ANCHORS.DLL and ANCHORC.DLL files.

defgroup= (obsolete)

This used to indicate which kind of database system is being used. It used to default to RCONN BTRS70, which refers to a server running the Pervasive/Btrieve system. As we are developing server versions for other database systems, alternate values can be specified here.

As of 2013-05-14, it is now obsolete and these entries will be ignored, and may be removed from WinPCS.INI.

Like the endpoint and server entries, defgroup can take a numerical suffix (defgroup1= defgroup2= etc) to correspond with the server of the same number.

dynamicserver=

There is the arrangement available when the WinPCS server does not sit on a fixed IP address. The reachout program runs there, and periodically updates the database within a dserver process running on a known IP-address, with the information about the WinPCS server's IP-address and port number. This entry indicates if this utility, named rqcli, is to be used to determine the server address. dynamicserver=off is the usual case where the WinPCS server is at a known IP address, and some string along the lines of dynamicserver=proj 1.2.3.4:56789 is used to specify that the dynamical IP address server is located at the specified IP-address (example here is 1.2.3.4) and port number (example 56789).

servertxt=

For the benefit of users of the Installer, each server identified as server= server1= server2= etc., can be named with servertxt= servertxt1= servertxt2= etc., so that the user of the Installer can get a list of understandable descriptions.

proxy=

Some systems may be distributed where there is a proxy server in use.

This setting determines if a proxy is to be used or not. There are three options:

proxy=no Do not use a proxy server, even if one might be present and being discovered.

proxy=autoDetermine if there is a proxy server available, and use it if one is found. This is the default behavior.

proxy=ipaddress:portUse the proxy with this IP address and port-number. Don't depend on other proxies that might be present.

Like the auth=, servertxt=, and endpoint= entries, there can be set one proxy1= proxy2= etc., differently for each server. The default will be the unnumbered proxy= entry for the ones that are not specified otherwise.

heartbeat=

heartbeat_ival_proxy=

heartbeat_ival_direct=

On some systems, to have the TCP connection stay open, it may need to be refreshed with some period. To turn this mechanism on, set heartbeat=on, to leave it off set heartbeat=off. The default is on. Turning it off is considered experimental, and does not always work all that well.

The values for the heartbeat_ival_...= entries are period times in seconds here. Depending on whether proxies are found, the period can be set differently. The default for the heartbeat_ival_proxy= is 10 seconds, and the default for the heartbeat_ival_direct= is 20 seconds. Generally, low values such as 10 or 20 s are useful on on long-haul connections, while heartbeat_ival_direct= periods as long as 3600 seconds may be useful on LANs.

auth=

This and its matching entry on the server-side server.INI file there controls authentication of the RPC calls. For Windows XP SP 2 these must be turned on for both server and client, for other variants of Windows they may be turned off. Set auth=on or auth=off accordingly.

The setting on the server is generally supposed to be the same. It is also possible to have different settings on the various servers, for this, we can use auth1=, auth2= and so on, for the corresponding server1=, server2= etc. See the discussion under errors 5 and 1747 .

This auth= setting is only used with DCERPC systems. TCP systems ignore it.

speedtest_interval_ms=

How frequent are speed-tests to be made, in milliseconds. Default 0, this means there are no speed tests being run.

speedtest_latest=

Small file to put the latest speed-test result into. This will be overwritten continually. Default is no file.

speedtest_logfile=

Log-file to put all the speed-test result into. Default is no file.

iolog=

Specifies a log-file, where the information between the client and server is shown. Useful for optimisations, but this file will grow.

interval=

This entry indicates the interval of checking the server presence and any messages it may have. The value given is in milliseconds. A typical value is 300000 milliseconds, or 300 seconds, that is, 5 minutes.

periodfiles=

This lists the files to find the periodcall function in. If this is missing or blank, the function will be expected to be in one of the files listed in the library entry instead.

periodcall=

This is the name of a periodically called function, called at every INTERVAL milliseconds. Its main purpose is to look for any messages for the user. If this entry is missing or refers to a blank function, nothing will be done.

rightmenu=

This indicates where the main source of the right-click menu entries in the hierarchies are. When set to rightmenu=DDMATRIX, the values in the field MENU_TXT in the DDMATRIX table are used, not the ones defined on the Strip.

lockout=

This can be used to temporarily stop users from logging in, or allow them to log in but after having been shown a message.

lockout=0
means the system is open and no message is to be shown. This is the default.

lockout=1 message
means the system is closed, and the message should explain why.

lockout=5 message
means the system is open, but there is some important message that should be shown.

editor=

Selects the text editor program that WinPCS invokes for editing INI and other text files. A typical value here would be editor=notepad.exe but other editors such as vim, ultraedit or textpad might be preferred.

ddesvc=

List the names of recognizeable DDE servers that WinPCS may recognize. There are also a need for a section for each of these programs. See the DDE-servers sections info below.

library=

Some globally loaded macrofunctions can be specified here in files.

appearance=

This entry allows changing the appearance of View, Entry, and Inspect screens, and the Report windows.

Set to exp1 for a moderately changed color scheme and new for a more radically different color scheme. Other values will give the old traditional look.

transferblock=

This entry defines the size of each data block used when transferring files to or from the server. Default value is 16384. Larger values make for better overall transfer speeds at the expense of less interactivity. On long-haul networks, where the latency is long, larger values such as 1048576 is preferrable. Powers of 2 are recommended but not mandatory.

onstart= and startfiles=

The onstart= defines an LCX function that is to be run when WinPCS is started. It is expected to be in one of the files listed in startfiles=. In addition to functions there, the functions in the library files will also be available.

debug=

This indicates whether serial-port or other debug-tracing is turned on or off. Values specify somewhere to go for the debugging output as well as trace( ) function calls from LCX code. These can be:

Debug=OFF
Debug=COMn
Debug=net ipaddress
Debug=net ipaddress:port
Debug=udp ipaddress
Debug=udp ipaddress:port
Debug=file filename

See also the discussions on the CWULIB( ) function and AimDebugOutput( ) function.

The net and udp options allow sending messages through a network . The default when no port is specified is 16385.

The COMn options select the numbered serial port.

The file puts the output into the named file, which will end up in the system32 or server directory.

cdatdebug=

When running the Transfer phase of CDAT, some functions in WinPCS are invoked, and if desired, the debugging-output can be turned on for this. as well, running in a separate channel from the main client debugging output described above.

The default is off, meaning no trace messages occur, and the various possible settings are the same as listed above for the debug= entry.

Note that any calls to the LCX function trace( ) from the CDAT-set callback functions will appear in this channel, not in any main client channel.

pdfcert_client_debug=

This controls debug-output for the PDFCert client interface (CPDCLI.DLL). The possible settings are the same as listed above for the debug= entry. The default is off for no output.

[MENU]

The entries in this section indicate the presence or absence of certain menu items. Each of the entries may have the PRESENT or ABSENT value.

The entries are: STDREP=
STDIMP=
EDITMENU=
RUNMENU=

[STRIP]

The entries here identify the items in the Strip, the row of icons at the bottom of the screen. The entries here have one of the forms:

name=L posn (menutext) (explanation) 
name=R posn (menutext) (explanation)

The name is the name of the symbol, defined in REFBMP31.INI and referenced as a drop-target fieldname in DDMATRIX.

L and R indicates if the icon is to be placed to the left or the right, and posn is the positioning from this side, where 0 is at the end.

menutext is the text that is to appear in the menu entries in the right-click menus in the hierarchies. If this is blank, this will not be included in these menus.

explanation is the text that will be shown as tool-tips on the main WinPCS screen, when the pointer is moved over one of the icons in the Strip. If this is blank, just with the empty pair of parentheses or entirely absent, there will be no tool-tips shown.

[PATHS]

Entries here are references to paths in various DLL and LCX files. Left side is some key value, right side is the path, which may be using the numbered drives.

Note that the numbered drives themselves are defined in the CLIENT.INI file, not in WinPCS.INI.

See Also:

Server-side [paths] in server.ini
ExpandPath( ) , ReducePath( ) , CLIENT.INI reference

[SYNFARE]

There are 2 entries here, QUOTE= and SEPARATOR= and they refer to the characters to be used for quote and field separators in SYNFARE CSV files. Record separators are always new-lines.

[PPVIEW]

The entries here are WRAPWIDTH and STARTWIDTH

WRAPWIDTH indicates how wide a column in a report may be before it defaults to a multiline layout. Applies to default report formatting. Typical value is WRAPWIDTH=60

STARTWIDTH determines how large the window should be when it opens. The value is the width of the window in percent of the screen horizontal area; about 80 looks good. This also changes the behavior of the landscape mode previews to extende fully horizontal instead of vertical.

A typical value thus is STARTWIDTH=80.

[HIER]

There can be three entries here, HIERWIDTH, SPACEMODE, and CASCADE

The entry HIERWIDTH indicates how wide the Hierarchy views are when they open. The value is in percentage of screen width. A typical value is 60 for 60% of the screen.

SPACEMODE can be 0, 1, or 2:

SPACEMODEDescription
0Strings in the hierarchies are shown with a single space character between the tag number and description. This is the default.
1Strings in the hierarchies are shown with a bit of space between the tag number and description. The amount of space varies slightly.
2Strings in the hierarches have the description part moved to the right, and all objects on the same level are indented the same amount, depending on the widest tag-number string. Subsequent levels are always indented further, even though the tag numbers here might not be that long.

CASCADE indicates if the hierarchies cascade down the same as Views and Reports do. Set to 1 to enable this, set to 0 to retain the old full-height at top behavior.

[VIEW]

There can be two entries here, WIDTH and ASPECT

The entry WIDTH, similar to HIERWIDTH for Hierarchies, indicates how wide the View windows are when they open. The value is in percentage of screen width. A typical value is 40 for 40% of the screen.

ASPECT indicates how tall the window is compared to the width. The number is a percentage, 100 makes a regular rectangular window, 160 makes it approximately square, 200 makes it tall.

These apply to View, Entry, and Inspect windows.

[REPORT]

FRAMEFILE= FRAMEFUNCTION= ORIENT=

The entries FRAMEFILE= and FRAMEFUNCTION= identifies the default report formatting function and the file where this function can be found. Orientation is set via ORIENT=, this is 0 for Portrait, 1 for Landscape, and 9 for asking the user for the one or the other.

STDTITLE= PRINTTITLE= SHORTTITLE=

STDTITLE= PRINTTITLE= SHORTTITLE= are titles to be shown on various reports.

SIERRMSG=

Entry SIERRMSG= controls if error-messages from bad tree-lines and other reasons for failure to run SI reports should appear as messages.

SIERRMSG=0 turns the message-box off.

SIERRMSG=1 turns the message-box on.

SICOUNT=

SICOUNT= defines the number of fields to pass for each iteration of status index reports. The default (whis is used by the siclient utility) is 2000. Lower values cause more frequent updates of the Odometer Window, but more passes on the network. Higher values cause less frequent, larger, amounts of data and a less frequent update rate.

PRLIST_INFO= PRLIST_FUNCTION=

PRLIST_INFO= sets the information text for the Print Certificates pushbutton in the Standard or Quick Reports.

PRLIST_FUNCTION= sets the name of the LCX function that is to perform this certificate printing. This function is similar to but not identical to the FORALL2 functions available in STAREP.INI. The files where this function is to be located should be listed in the entry PRLIST= under section [LCX] in client.ini.

TEMPFILES=

TEMPFILES= sets the location where the temporary file which is used for transferring to Excel is to be written. The argument is some path, either a complete system path, or a "numbered-drives" path. If this entry is missing or blank, the default temporary file location that is set in the operating system will be used.

ADDBLANK=

ADDBLANK=yes will make the empty reports show with a record and the text "No records found" in the first column.

ADDBLANK=no will cause empty reports to not be shown at all.

PDFOUTPUT=

PDFOUTPUT= sets where the saved PDF files from immediate printing of reports are to go. Default is 9:\reportpdfs Numerical paths are used here, as there is an internal call to the equivalent of ExpandPath( ).

See also: Quick Report Print List Functions

[RCONN]

files= is for the client-side make and break activities functions.

stadlg= are files with status-dialog functions in them. These are also referenced for CDAT's barcodediscovery= function

[HIERTOP]

Entries here give parameters for various hierarchies. They are on one of the the forms:

table=(Title) tail-arguments
phase table=(Title) tail-arguments

The optional phase can allow hierarchies off the same level to display and act differently on subordinate items, by adjusting the tail-arguments accordingly.

The tail-arguments are a sequence of space-separated arguments, each one can be one of these:

E:table Stop opening more levels of connections when double-clicking on some object in this table. Open this table, but do not open further levels.
L:table1+field1==table2+field2 On double-clicking to open any object from either table1 or table2, make sure the field1 in table1 and field2 in table2 have the same value when one of them is a parent item of the other. Only display those items where these fields are the same.
S:table Issue a status-dialog when double-clicking on some object in this table, instead of opening more connections.
X:table Do not open or display objects from this table whenever encountered.
Y:table1>table2 When double-clicking on objects of table1, do not open up or show the attached objects from table2, but do display others.
C:fieldname+value+value+... Suppress connections where the fieldname in the connection table has one of the values shown in the list.

[dde server name]

REGEDIT=

EXTENSION=

There is one of these for each of the servers listed in the ddesvc entry. In addition, for EXCEL, the following extra options exist:

[EXCEL]
ROWLETTER=R
COLLETTER=C

Here standard Excel idenfies cells as RrownoCcolno, but other language versions are known to have different letters. German uses Z and S, Portuguese has F and C, and Polish apparently use W and K. To allow use with different languages, the various alternate sets can be listed here, as for example:

ROWLETTER=RZFW
COLLETTER=CSCK

Now these can be R and C, Z and S, F and C, or W and K.
