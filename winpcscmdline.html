WinPCS Client Command-Line

File Date: 2012-05-04

The WinPCS client command-line should be the invocation of the WINPCS.EXE program, with the working directory being SYSTEM32. This is the client-side system directory, where the AnchorC.DLL project-specific DLL is.

The server may be on the other side of a VPN inside a WAN, in this case, the server-side directories DATA and SERVER do not need to be copied across to the client. Only the client-side directories such as SYSTEM32, INI, and FORMS are required locally. With this division, no disk mapping needs to be done to the server for WinPCS to operate.

-U username

This argument forces the specified user-name to be entered rather than the network-login name.

-L

This argument present forces a login-dialog, even if this is not otherwise required. If passwords are enabled via Anchor string 601, this dialog will always appear.

-S servernumber

This allows easy selection of alternate servers, without having to maintain different system directories for them. The default server is the one shown as server=... in WinPCS.INI, the others may be identified as server1=... server2=... and so on, they are selected via -S 1 or -S 2 on the command-line.

-D directory

This argument allows the working directory, where the ANCHORC.DLL file is, to be specified on the command-line.

-N debug-command

This argument allows the client to emit debugging information to a listening netrx server, udprx server, or a COM port. The debug-command is the same as used with the DEBUG= entry in WinPCS.INI.

Example

-N "udp 192.168.2.107:16381"

will make this client transmit debug text to a udprx listener on the machine 192.168.2.107 port 16381, even though the standard DEBUG= setting in WinPCS.INI is different. The quotes around the string ensure that the space between net and the IP-address becomes part of the argument.
